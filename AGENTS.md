# AGENTS.md

## Skill routing

- Use "orchestrator" for multi-layer or unclear tasks
- Use "software-architect" for system architecture
- Use "backend-architect" for APIs and business logic
- Use "database-optimizer" for schema, indexes, and query performance
- Use "frontend-developer" for frontend implementation and API integration
- Use "ui-ux-designer" for dashboards, forms, tables, and UX
- Use "security-engineer" for auth, permissions, and security hardening
- Use "devops-automator" for environments, CI/CD, deployment, and monitoring

## Rules

- Prefer production-ready solutions
- Avoid generic answers
- Keep backend as source of truth
- Highlight risks and trade-offs
